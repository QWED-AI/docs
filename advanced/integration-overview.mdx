## вҡ пёҸ Common mistake

**You might think:**



### вқҢ DON'T DO THIS:

```python
# вқҢ WRONG!
import openai
from qwed import QWEDClient

# Calling LLM yourself
response = openai.ChatCompletion.create(...)

# Then trying to verify
qwed.verify(response.content)  # TOO LATE!
```

**Why this fails:**
- рҹҡ« No control over LLM prompts
- рҹҡ« No DSL enforcement  
- рҹҡ« Vulnerable to prompt injection
- рҹҡ« Can't guarantee structured output

---

## вң… Correct approach



### вң… DO THIS:

```python
# вң… CORRECT!
from qwed import QWEDClient

qwed = QWEDClient(api_key="qwed_...")

# Just call QWED directly
result = qwed.verify("Is 2+2 equal to 4?")

print(result.verified)  # True вң…
```

**Why this works:**
- вң… QWED controls LLM internally
- вң… Structured prompts ensure DSL output
- вң… Formal verification layer active
- вң… Deterministic results

## рҹ“– Quick start examples

### 1пёҸвғЈ Math verification

```python
from qwed import QWEDClient

client = QWEDClient(api_key="your_key")

# вң… Natural language input
result = client.verify("Is 2+2 equal to 5?")

# What happens inside QWED:
# рҹ“қ LLM extracts: "2+2=5"
# рҹ”¬ SymPy verifies: 2+2 = 4 (not 5!)
# вқҢ Returns: verified=False

print(result.verified)  # False
print(result.reason)    # "Expected 4, got 5"
print(result.evidence)  # {"calculated": 4, "claimed": 5}
```

**Visual Flow:**
```
User Query вҶ’ QWED вҶ’ [LLM: "2+2=5"] вҶ’ [SymPy: 4вү 5] вҶ’ вқҢ Failed
```

---

### 2пёҸвғЈ Code security

```python
dangerous_code = """
def get_user(username):
    query = f"SELECT * FROM users WHERE name='{username}'"
    return db.execute(query)
"""

result = client.verify_code(dangerous_code, language="python")

# What happens inside QWED:
# рҹ“қ LLM identifies: String interpolation in SQL
# рҹ”¬ AST parser finds: User input in query
# рҹҡ« Security engine: SQL INJECTION RISK
# вқҢ Returns: blocked=True

print(result.blocked)  # True рҹҡ«
print(result.vulnerabilities)  # ["SQL Injection"]
print(result.severity)  # "HIGH"
```

**Visual Flow:**
```
Code вҶ’ QWED вҶ’ [LLM: Detects SQL] вҶ’ [AST: f-string in query] вҶ’ рҹҡ« BLOCKED
```

## рҹҺҜ Do's and don'ts

### вң… DO:

```python
# вң… Call QWED directly
result = qwed.verify("Calculate 15% of 200")

# вң… Use natural language
result = qwed.verify("Is the square root of 16 equal to 4?")

# вң… Let QWED handle LLM internally
result = qwed.verify_code(untrusted_code, language="python")

# вң… Trust the verification results
if result.verified:
    use_output(result.value)
```

### вқҢ DON'T:

```python
# вқҢ Call LLM yourself first
llm_output = openai.chat(...) 
qwed.verify(llm_output)  # TOO LATE!

# вқҢ Try to bypass QWED's LLM
result = qwed.verify_math("2+2", skip_llm=True)  # No such option

# вқҢ Mix QWED calls with direct LLM calls
llm_result = gpt4.complete(...)
qwed_result = qwed.verify(...)  # Inconsistent!

# вқҢ Assume LLM output is correct
value = llm.generate("Calculate...")
use_value_directly(value)  # DANGEROUS!
```

## рҹҺү Quick summary

### Remember these 3 things

1. **вқҢ Don't call LLM yourself**  
   Let QWED handle it internally

2. **вң… Call QWED directly**  
   Use natural language queries

3. **рҹ”’ Trust the verification**  
   QWED uses formal methods, not guessing

### One-line integration

```python
result = QWEDClient(api_key="...").verify("Your question here")
```
