## v4.0.1 тАФ Sentinel Guard Sync
**Released: March 23, 2026** ┬╖ [GitHub Release](https://github.com/QWED-AI/qwed-verification/releases/tag/v4.0.1)

Syncs the TypeScript SDK, backend API schemas, and security fixes identified during the v4.0.0 rollout.

### New endpoints

- **`POST /verify/process`** тАФ Glass-box reasoning process verifier with IRAC structural compliance and custom milestone validation. See [API reference](/api/endpoints) and [Process Verifier engine](/engines/process).
- **`POST /verify/rag`** тАФ Document-Level Retrieval Mismatch (DRM) defender for RAG pipelines. See [API reference](/api/endpoints).
- **Agent security checks** тАФ `POST /agents/{id}/verify` now accepts `security_checks` with `exfiltration` and `mcp_poison` options to run `ExfiltrationGuard` and `MCPPoisonGuard` before verification. See [API reference](/api/endpoints).

### Security fixes

- **Information disclosure** тАФ Removed raw exception details from `/verify/rag` error responses. Exceptions are now redacted via `redact_pii()` and clients receive only a generic `INTERNAL_VERIFICATION_ERROR` message.
- **Symbolic precision** тАФ `RAGVerifyRequest.max_drm_rate` changed from `float | str` to `str` only, with a `field_validator` enforcing `Fraction`-compatible values (e.g. `"0"`, `"1/10"`).

### SDK changes (`@qwed-ai/sdk@4.0.1`)

- **`verifyProcess()`** тАФ Validates AI reasoning traces using IRAC or custom milestone lists. See [TypeScript SDK](/sdks/typescript).
- **`verifyRAG()`** тАФ `maxDrmRate` type changed from `number` to `string` for symbolic precision.
- **`verifyAgent()`** тАФ Returns `AgentVerificationResponse` with payload aligned to backend schema. Agent IDs are URL-encoded.
- **Type fixes** тАФ `VerificationResultData.risk` and `risk_level` separated. Added `Process`, `RAG`, `Security` to `VerificationType` enum.

## v4.0.0 тАФ Sentinel Edition ЁЯЫбя╕П
**Released: March 12, 2026** ┬╖ [GitHub Release](https://github.com/QWED-AI/qwed-verification/releases/tag/v4.0.0) ┬╖ [PyPI](https://pypi.org/project/qwed/4.0.0/)

> **147 commits** since v3.0.1 тАФ the largest update in QWED history.

### ЁЯФз v4.0.0 patch тАФ Provider fallbacks, .env loading, and Gemini stability

- **Native Gemini provider** тАФ Gemini now runs through a dedicated provider with native support for math, logic, stats, fact, and image verification. All API calls enforce a 30-second timeout and automatically strip Markdown code fences from responses.
- **Centralized .env loading** тАФ Environment variables now load in a deterministic priority order: project `.env` first, then `~/.qwed/.env`. This prevents configuration drift between CLI and server contexts.
- **Gemini AST stability** тАФ Fixed intermittent JSON parse failures when Gemini returns code-fenced responses during logic translation.

### ЁЯЖХ Agentic Security Guards (Phase 17)

A brand-new guard subsystem for securing AI agent tool chains and RAG pipelines:

- **RAGGuard** тАФ Detects prompt injection, data poisoning, and context manipulation in RAG pipelines. IRAC-compliant reporting.
- **ExfiltrationGuard** тАФ Prevents data exfiltration through agent tool calls by analyzing output patterns and destination validation.
- **MCP Poison Guard** тАФ Detects poisoned or tampered MCP tool definitions before agent execution.

All three guards went through **five rounds** of security review via CodeRabbit and SonarCloud.

### ЁЯЖХ New Standalone Guards

- **SovereigntyGuard** тАФ Enforces data residency policies and local routing rules (GDPR, data localization).
- **ToxicFlowGuard** тАФ Stateful detection of toxic tool-chaining patterns across multi-step agent workflows.
- **SelfInitiatedCoTGuard (S-CoT)** тАФ Verifies self-initiated Chain-of-Thought logic paths for reasoning integrity.

### ЁЯЖХ Process Determinism

A new class of deterministic verification:

- **ProcessVerifier** тАФ IRAC/milestone-based process verification with decimal scoring, budget-aware timeouts, and structured compliance reporting. Ensures AI-driven workflows follow deterministic process steps тАФ not just correct answers, but correct *procedures*.

### ЁЯФТ Critical Security Fixes

- Replaced all `eval()` with AST-compiled execution (Code Injection Prevention).
- Patched critical sandbox escape and namespace mismatch.
- Hardened SymPy input parsing against injection.
- Fixed URL whitespace bypass and protocol wildcard bypass.
- Resolved CVE-2026-24049 (Critical), CVE-2025-8869, and HTTP request smuggling.
- Fixed all 19 Snyk Code findings.
- Secured exception handling across `verify_logic`, `ControlPlane`, `verify_stats`, `agent_tool_call`.

### ЁЯР│ Docker Hardening
