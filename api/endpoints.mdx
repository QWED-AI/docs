## Verification Endpoints

### POST /verify

Auto-detect and verify any claim.

**Request:**

```json
{
  "query": "Is 2+2=4?",
  "options": {
    "include_attestation": true
  }
}
```

**Response:**

```json
{
  "status": "VERIFIED",
  "verified": true,
  "engine": "math",
  "verification_mode": "SYMBOLIC",
  "result": {
    "is_valid": true
  },
  "attestation": "eyJhbGciOiJFUzI1NiIs..."
}
```

> **`verification_mode`**: Indicates whether the result was proven by a deterministic solver (`SYMBOLIC`) or required LLM fallback (`HEURISTIC`). See [Determinism Guarantee](/advanced/determinism-guarantee) for details.

### POST /verify/math

Verify mathematical expressions.

```json
}
```

### POST /verify/logic

Verify logical constraints.

**Request:**

```json
{
  "query": "(AND (GT x 5) (LT y 10))",
  "format": "dsl"
}
```

**Response:**

```json
{
  "status": "VERIFIED",
  "verified": true,
  "verification_mode": "SYMBOLIC",
  "provider_used": "openai_compat",
  "result": {
    "satisfiability": "SAT",
    "model": {"x": 6, "y": 9}
  }
}
```

The `provider_used` field indicates which LLM provider handled the translation step. This field is included in both success and error responses, which helps with debugging provider routing issues.

---

### POST /verify/code

Check code for security vulnerabilities.

**Request:**

```json
{
  "code": "import os\nos.system('rm -rf /')",
  "language": "python"
}
```

**Response:**

```json
{
  "status": "BLOCKED",
  "verified": false,
  "verification_mode": "SYMBOLIC",
  "result": {
    "vulnerabilities": [
      {
        "type": "os.system",
        "severity": "critical",
        "line": 2,
        "message": "Shell command execution"
      }
    ]
  }
}
```

### POST /verify/fact

Verify factual claims against a provided context.

**Request:**

```json
{
  "claim": "Paris is the capital of France",
  "context": "France is a country in Western Europe. Its capital is Paris.",
  "provider": "openai"
}
```

**Response:**

```json
{
  "status": "VERIFIED",
  "verified": true,
  "engine": "fact",
  "result": {
    "is_valid": true,
    "confidence": 0.98,
    "message": "Claim is supported by the provided context"
  }
}
```

---

### POST /verify/consensus

<Info>New in v4.0.0</Info>

Multi-engine consensus verification. Runs the query through multiple verification engines and requires agreement above a confidence threshold.

**Request:**

```json
{
  "query": "The square root of 144 is 12",
  "verification_mode": "high",
  "min_confidence": 0.8
}
```

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `query` | string | Yes | The claim to verify |
| `verification_mode` | string | No | `single`, `high`, or `maximum` (default: `single`) |
| `min_confidence` | float | No | Minimum confidence threshold, 0–1 (default: `0.8`) |

**Response:**

```json
{
  "status": "VERIFIED",
  "verified": true,
  "engine": "consensus",
  "result": {
    "confidence": 0.95,
    "engines_used": ["math", "logic"],
    "agreement_rate": 1.0
  }
}
```

### POST /verify/image

<Info>New in v4.0.0</Info>

Verify claims about image content. Accepts multipart form data with an image file (max 10 MB).

**Request:**

```bash
curl -X POST https://api.qwedai.com/v1/verify/image \
  -H "X-API-Key: qwed_your_key" \
  -F "image=@photo.jpg" \
  -F "claim=This image contains a cat"
```

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `image` | file | Yes | Image file (max 10 MB) |
| `claim` | string | Yes | The claim about the image to verify |

**Response:**

```json
{
  "status": "VERIFIED",
  "verified": true,
  "engine": "image",
  "result": {
    "is_valid": true,
    "confidence": 0.92,
    "message": "Claim is consistent with image content"
  }
}
```

### POST /verify/batch

Verify multiple items at once.

**Request:**

```json
{
  "items": [
    {"query": "2+2=4", "type": "math"},
    {"query": "3*3=9", "type": "math"},
    {"query": "(GT x 5)", "type": "logic"}
  ],
  "options": {
    "parallel": true,
    "stop_on_failure": false
  }
}
```

**Response:**

```json
{
  "results": [...],
  "summary": {
    "total": 3,
    "verified": 3,
    "failed": 0,
    "success_rate": 100.0
  }
}
```

## Agent endpoints

### POST /agents/register

Register a new AI agent with QWED for verified agentic workflows.

**Request:**

```json
{
  "name": "MyAgent",
  "agent_type": "supervised",
  "description": "Financial analysis agent",
  "permissions": ["math", "logic", "code"],
  "max_cost_per_day": 100.0
}
```

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `name` | string | Yes | — | Agent display name |
| `agent_type` | string | Yes | — | Agent type (e.g., `supervised`, `autonomous`) |
| `description` | string | No | — | Agent description |
| `permissions` | array | No | `[]` | Allowed verification engines |
| `max_cost_per_day` | float | No | `100.0` | Daily budget cap in USD |

**Response:**

```json
{
  "agent_id": "agent_abc123",
  "token": "qwed_agent_...",
  "status": "registered"
}
```

---

### POST /agents/{agent_id}/verify

Verify a claim using an agent token. Creates an auditable record tied to the agent.

**Request:**

```json
{
  "query": "2+2=4",
  "provider": "openai"
}
```

**Headers:**

```text
Authorization: Bearer qwed_agent_...
```

---

### POST /agents/{agent_id}/tools/{tool_name}

Submit an agent tool call for risk evaluation before execution.

**Request:**

```json
{
  "tool_params": {
    "query": "SELECT * FROM users",
    "dialect": "postgresql"
  }
}
```

---

## Observability endpoints

<Info>New in v4.0.0</Info>

### GET /metrics

Returns global system metrics (request counts, latencies, engine usage).

### GET /metrics/{organization_id}

Returns per-tenant metrics scoped to a specific organization.

### GET /metrics/prometheus

Returns metrics in Prometheus scrape format for integration with monitoring infrastructure.

### GET /logs

Returns verification logs for the authenticated tenant. Supports pagination.

## Admin endpoints

<Info>New in v4.0.0</Info>

These endpoints require the `admin:all` API key scope.

### GET /admin/compliance/export/csv

Export the full audit trail as a CSV file.

### GET /admin/compliance/verify/{log_id}

Cryptographically verify a specific audit log entry using HMAC and hash chain validation.

### GET /admin/compliance/report/soc2/{org_id}

Generate a SOC 2 Type II compliance report for an organization.

### GET /admin/security/threats/{org_id}

Returns a real-time threat summary for an organization, including blocked injection attempts and anomalous patterns.

### POST /admin/keys/rotate

Rotate an API key. Available to admin and member roles.

**Request:**

```json
{
  "key_id": "key_abc123"
}
```
