The **Schema Verifier** goes beyond standard JSON validation. It combines **Pydantic** structure enforcement with **Symbolic Math** checks deeply embedded within the schema.

## How It Works

It validates that:

1. **Structure:** The output matches the required JSON keys and types.
2. **Logic:** The numeric values _inside_ the JSON are mathematically consistent (e.g., `total == sum(items)`).

## Usage

```python
schema = {
    "type": "object",
    "properties": {
        "items": {"type": "array", "items": {"type": "number"}},
        "total": {"type": "number"}
    },
    # QWED Extension: Math Logic
    "qwed:constraints": [
        "total == sum(items)"
    ]
}

response = client.verify_schema(
    obj={"items": [10, 20], "total": 30},
    schema=schema
)
# -> ✅ Verified

response = client.verify_schema(
    obj={"items": [10, 20], "total": 300}, # LLM hallucinations
    schema=schema
)
# -> ❌ MATH ERROR: total (300) != sum(items) (30)
```

## `uniqueItems` validation

When your schema includes `"uniqueItems": true`, the engine checks that every element in the array is distinct. If the engine cannot complete the uniqueness check — for example, because an item triggers a `TypeError` or `ValueError` during comparison — it reports a `uniqueness_validation_error` instead of silently passing. This fail-closed behavior ensures that unverifiable arrays are never treated as valid.

```python
schema = {
    "type": "object",
    "properties": {
        "tags": {
            "type": "array",
            "uniqueItems": True,
            "items": {"type": "string"}
        }
    }
}

# Duplicate items are caught
response = client.verify_schema(
    obj={"tags": ["billing", "billing"]},
    schema=schema
)
# -> ❌ uniqueness_violation: Array contains duplicate items

# Items that cannot be compared deterministically also fail
# -> ❌ uniqueness_validation_error: uniqueItems could not be verified deterministically
```

<Info>
  This fail-closed behavior was introduced in v5.0.6. Earlier versions silently skipped the uniqueness check when items could not be compared. See the [changelog](/changelog#v506--schema-uniqueitems-fail-closed) for details.
</Info>

## When to use

- **Invoice Processing:** Ensure line items sum to the total.
- **Financial Reports:** Ensure balance sheets balance.
- **Tax Forms:** Ensure calculated fields match underlying data.
