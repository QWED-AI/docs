The Stats Engine executes statistical queries on tabular data using a secure Docker sandbox.

<Warning>
**Changed in v4.0.4:** The Wasm and restricted in-process execution fallbacks have been removed. Statistical verification now requires the secure Docker sandbox. If Docker is unavailable, the engine returns a `503` error instead of falling back to in-process execution.
</Warning>

## Features

- **Docker Sandbox** - Full isolation for all statistical code execution
- **Security Validation** - Pre-execution AST-based code analysis
- **Fail-closed design** - No in-process fallbacks when Docker is unavailable

## Usage

```python
import pandas as pd
from qwed_sdk import QWEDClient

client = QWEDClient(api_key="qwed_...")

# Create sample data
df = pd.DataFrame({
    "product": ["A", "B", "C"],
    "sales": [100, 200, 150]
})

# Verify statistical claim
result = client.verify_stats(
    query="What is the average sales?",
    data=df
)
print(result.answer)  # 150.0
```

## Execution requirements

Statistical verification runs model-generated Python code exclusively inside a Docker container. You must have Docker available in your deployment environment.

| Requirement | Details |
| ----------- | ------- |
| Docker daemon | Must be running and reachable by the QWED process |
| Health check | The executor pings Docker on every request to confirm availability |
| Failure mode | Returns HTTP 503 when Docker is unreachable |

<Note>
The Wasm and restricted in-process executors are retained in the codebase as disabled stubs. They always return a failure result and never execute user-supplied code.
</Note>

## Direct Operations

For simple operations, bypass code generation:

```python
result = client.compute_statistics(
    data=df,
    column="sales",
    operation="mean"  # mean, median, std, var, sum, count, min, max
)
```
